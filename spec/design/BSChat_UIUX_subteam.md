## 1. 產品規則（畫面必須講清楚）

| 規則 | UI 呈現 |
|------|---------|
| 共享範圍＝**僅所屬子團隊** | PrivacyStrip／空狀態寫「僅子團隊可見」 |
| **誰可建隊（拍板建議）** | **企業成員皆可建**（建隊者＝負責人）；**主 Admin 可在企業後台看／解散任何子團隊**。Later：Admin 政策「僅指定成員可建」 |
| 企業成員但**未加入任何子團隊** | 名片庫／搜尋＝**僅自己的卡** |
| 不做單張共享 | 詳情頁**無**私有／共享開關 |
| ≠ AI 公開推薦 | 文案分離 |

### 1.1 誰可建隊 — 產品／市場／企業角度（決策依據）

| 角度 | 判斷 |
|------|------|
| **產品** | 主管要立刻帶兩隊；若每次找 Admin 開權限，dogfood 卡死。→ 預設**可自助建隊**。 |
| **市場** | 對標 Slack／Teams 頻道：先鬆、企業要緊再收。賣點寫「多業務線互不串味」，不是「完整 HR 組織樹」。 |
| **企業採購** | 買家怕失控。→ 必須給主 Admin **總覽＋強制解散**；Next 再加「限制誰能建隊」政策開關。 |

**結論（本輪）**：全員可建 + Admin 治理；不做職稱系統。  
**方案**：子團隊能力 **僅企業**（見 PRD §4／DDR-v4-11）。  
**平台後台**：SaaS Platform Admin＝Next（DDR-v4-12），與租戶 `/admin/org` 分開規劃。

## 2. 資訊架構與路由

### 2.1「我的」改分頁（避免長捲）

固定頂部：帳號卡（姓名／Email／方案）＋ **Segment 分頁**（不整頁往下堆）。

| Tab | Free／Pro | 企業 |
|-----|-----------|------|
| **總覽** | 用量、公開推薦剩餘、升級 CTA、PWA、登出 | 同上＋企業後台入口（Admin） |
| **子團隊** | **不顯示**（或鎖住＋「升級企業版」一句） | 隊列表、建立、進入 |
| **偏好** | 精準度、隱私說明 | 精準度、Pro／企業自動更新、隱私說明 |

舊「一般團隊邀請」在企業路徑下併入子團隊；Free／Pro 不開組織共享池（對齊 DDR-v4-11）。

| 路由／入口 | 誰 | 內容 |
|-----------|----|------|
| `/settings` | 全員 | 分頁殼（預設總覽；`?tab=teams` 直達子團隊） |
| `/teams/new`（sheet） | 企業成員 | 建立子團隊 |
| `/teams/[id]` | 隊員／負責人 | 成員、邀請、離開／解散 |
| `/join/team/[token]` | 受邀者 | 加入子團隊 |
| `/admin/org` 擴充 | 主 Admin | Tab「子團隊」：全公司隊列表、人數、解散 |
| `/contacts` | 全員 | Chip／隊名；無隊＝個人庫 |

**不做本輪**：跨隊轉卡、職稱系統、單張共享、完整組織樹。

## 3. 關鍵畫面說明

### 3.1 我的 · 分頁殼
- 頂部帳號卡固定（不隨 Tab 消失）
- Segment：`總覽`｜`子團隊`｜`偏好`（一屏內切換，不長捲堆疊）
- **總覽**：用量、公開推薦剩餘、升級／企業後台、PWA、登出
- **子團隊**：隊列表、「＋ 建立」、空狀態「尚未入隊＝個人庫」
- **偏好**：精準度、Pro 自動更新、隱私說明

### 3.2 建立子團隊
- 欄位：隊名（必填，例「業務一組」）、說明（選填）
- 送出後進隊詳情；建立者標「負責人」
- 成功 toast：「已建立 · 邀請同事加入後才會共享」

### 3.3 子團隊詳情
- Header：隊名、公司名、人數
- 成員列表：頭像縮寫、姓名、Email、標籤「負責人」
- CTA：「邀請成員」（複製連結／分享）
- 負責人：可移除成員、解散團隊（二次確認）
- 一般成員：「離開子團隊」（離開後只剩自己的庫）

### 3.3b 企業後台 · 子團隊（Admin 治理）
- 表：隊名、負責人、人數、建立日、操作「解散」
- 說明：解散後成員回個人庫；名片不刪

### 3.4 邀請加入
- 文案：「加入 {公司} · {隊名}」
- 與企業席次邀請分開（企業＝進租戶；子團隊＝進共享範圍）

### 3.5 名片庫
- 頂部 PrivacyStrip：「僅子團隊可見 · 不對外公開」
- 有隊時 chip 可顯示目前隊名；列項次要行「由 X 收錄 · 業務一組」
- 無隊：strip「個人庫 · 加入子團隊後可與同隊共享」＋連結去「我的 → 子團隊」

## 4. 文案表

| 位置 | 文案 |
|------|------|
| PrivacyStrip（有隊） | 僅子團隊可見 · 不對外公開搜尋 |
| PrivacyStrip（無隊） | 個人庫 · 尚未加入子團隊 |
| 建立成功 | 已建立。邀請同事後，名片才會在隊內共享 |
| 離開確認 | 離開後你將看不到同隊其他人收錄的名片（自己的仍在） |
| 解散確認 | 解散後全隊成員改回個人庫；名片不會刪除 |
| 與 AI 推薦區隔 | （設定隱私區）團隊共享 ≠ 允許 AI 推薦 |
