> **文件性質**：2026-07-14 產品重審後的**終局定位**；收斂 v3 的「團隊情報」為**私有／dogfood 層**，並把**可選公開的電子名片網絡**拉回主軸。  
> **確認日期**：2026-07-14  
> **狀態**：已確認（用戶「繼續吧」）  
> **上游**：`spec/product/BSChat_network_wedge.md`（已拍板）、`screen-4-plan-search.html`、`screen-1-opportunity-briefing.html`  
> **與舊版關係**：v1/v2 核心價值與負面情境 N-01~08、DDR-5/6/7 **延續**；v3 的 persona／團隊池／dossier **保留為 Layer A**；v3 §6「公開池＝Later」**由本版覆寫**。  
> **一句話**：紙名片與電子名片都變成可被 AI 讀懂的資產；需求出現時，AI 以可見的 Plan 搜你的庫，並（在允許時）推薦自願公開的電子身份。

## 0. 為什麼有 v4

| 觀察 | 結論 |
|------|------|
| 產品「感覺像名片管理、AI 很虛」 | 差異多在後端；搜尋像列表，缺少**看得見的 AI 過程**與**庫外網絡** |
| 理想＝電子名片社交／網絡 | **不是**動態牆／聊天／互加好友，而是**可選公開 → 被 AI 推薦** |
| v3 團隊情報 | 仍必要：私有供給、dogfood、助理表格痛；但是**終點不夠** |

**品類定位**：商務關係的外接大腦 ＋ **可發現的電子身份網絡**（非 CRM、非 IM、非動態社群）。

## 1. Persona

延續 v3 團隊角色（主管／業務／業務助理／採購），並加上：

| 角色 | 在 v4 的額外目標 |
|------|------------------|
| **企業 Admin** | 發布／管理員工**電子名片**，決定誰可「允許 AI 推薦」 |
| **Pro 個人／團隊業務** | 需求當下除了自己的庫，還能看到**公開推薦** |
| **Free** | 用自己的庫 + Plan 體感；公開推薦僅**終身試用 2 次** |

**首要驗證（仍優先 dogfood）**：助理是否少開表、採購是否少先開 Google、搜尋是否「感覺有 AI」。  
**網絡層驗證（其後）**：企業是否願意公開；Pro 是否因公開推薦而升級／回訪。

## 2. 核心價值

### 2.1 殺手時刻（Plan + 簡報）
需求一句話 → **固定五步 Plan（可見）** → 分區結果 → **商機簡報**（理由／沉睡／開場白／不硬湊）。  
原型：`screen-4-plan-search.html`、`screen-1-opportunity-briefing.html`。

### 2.2 兩層供給
| 層 | 名稱 | 說明 |
|----|------|------|
| **A · 私有／團隊池** | 你的名片庫 | 掃入紙名片、團隊共享；永遠不跨公司外洩 |
| **B · 公開推薦池** | 電子名片 opt-in | 僅企業可開「允許 AI 推薦」；Pro／企業可搜；Free 終身試用 2 次 |

### 2.3 電子名片
可分享（連結／QR）；身份持續存在。  
**必填**：姓名、公司、職稱、對外連結。  
**選填**：職責／產品關鍵字、頭像、一句話介紹。  
**公開展示禁止**：電話、Email（聯絡走外鏈）。

### 2.4 AI 自動 dossier（延續 v3，範圍務實）
公司產品（M6）+ 個人職責推估（M3）為 P0。  
特質／經歷等「助理表欄」以**已有能力＋人工備註**為主，不強制一次做完三欄編造。

### 2.5 團隊共享池（延續 v3）
公司內預設共享；v1 不建完整權限 UI。

---

## 3. 可見與隱私模型（三層）

1. **跨帳號私有收錄**：永遠不可被他人搜到（寫死）。  
2. **公司內團隊池**：政策可設（v1 預設共享、無複雜 UI）。  
3. **公開推薦**：僅電子名片 **opt-in**；未勾選 → 永不進公開池。  

Plan／文案不得暗示「正在搜全平台私人抽屜」。  
公開池空或無權限 → 誠實略過或說明，只回庫內結果。

## 5. Plan 模式（搜尋 UX · P0）

固定五步（可略過第 3 步並標示）：

1. 理解你的需求  
2. 在你的名片庫搜尋  
3. 在公開可推薦池查找  
4. 比對誰最符合  
5. 整理商機簡報  

原則：真步驟、可打勾；禁止假進度條。  
結果分區：**你的名片庫**｜**AI 推薦 · 公開身份**。

## 6.6 企業租戶（方案 B · 完整 · 已鎖定 2026-07-16）

> 權威：`spec/product/BSChat_enterprise_tenant_model_B.md`

**一句話**：企業＝可治理租戶；**唯一主 Admin** 發**企業席次邀請**控名單；進租戶＝enterprise（B1）；一般人不能自助變企業。

**主流程**
1. 申請企業版 → 平台核准（dogfood／業務測可用指令開通「本公司」）  
2. 指定主 Admin（每租戶 1 人；可轉移）  
3. Admin 邀請成員 → 對方註冊／登入後入租戶並得企業能力  
4. Admin 管電子名片；成員可草稿；**允許 AI 推薦僅 Admin**  
5. Admin 可撤銷邀請、移除成員（失去企業能力、公開下架）

**Dogfood／本公司業務測**：必須能不靠工程改 DB，由 Admin 邀業務進來測團隊池＋公開供給。

**明確不做（本輪）**：無審核自助開企業、次 Admin、SSO、正式金流（核准代替付款）。

## 7. User Stories（摘要）

**US-v4-1（Plan 體感）**  
作為使用者，我要看到 AI 的搜尋計劃步驟，這樣我知道系統在做什麼、不是普通搜尋框。  
AC: Given 送出問句, When 搜尋進行, Then 依序顯示五步狀態（含可略過的步驟 3）。

**US-v4-2（庫內商機）**  
作為業務／採購，我要從團隊池反向找到人並拿到簡報與開場白。

**US-v4-3（公開推薦）**  
作為 Pro，我要在庫不足時看到「AI 推薦 · 公開身份」，且不含他人電話 Email。

**US-v4-4（企業曝光）**  
作為企業主 Admin，我要為電子名片開啟／關閉「允許 AI 推薦」。

**US-v4-4b（企業邀請控名單）**  
作為企業主 Admin，我要邀請／撤銷／移除成員，這樣只有我允許的人能代表本公司。  
AC: Given 企業租戶已開通, When Admin 發企業邀請且對方完成註冊或登入, Then 對方入租戶且具備企業能力（B1）；撤銷後連結失效；移除後失去企業能力且公開名片下架。

**US-v4-4c（業務 dogfood）**  
作為本公司業務，我接受 Admin 邀請後能進企業租戶測團隊池與搜尋，無需工程改資料庫。

**US-v4-5（Free 試用）**  
作為 Free，我只能終身試用公開推薦 2 次；用完後步驟 3 略過並引導升級；不月重置。

**US-v4-6（隱私）**  
作為任何人，我的未公開名片與私有收錄永不被外人搜到。

（v3 US 助理 dossier／團隊搜尋等仍然有效，改編號對照即可。）

## 8. 成功指標

| 階段 | 指標 |
|------|------|
| Dogfood | 助理少開表、採購少先 Google、搜尋滿意度、沉睡促成接觸 |
| 體感 | 使用者能描述「AI 有計劃步驟」（質性） |
| 網絡 | 企業公開 opt-in 數、Pro 公開推薦使用率、Free→Pro 因試用轉換 |

## 9. 明確不做

- 聊天、動態牆、追蹤、互加好友式社交  
- 未 opt-in 名片進入公開池  
- 公開池展示電話／Email  
- Free 公開推薦月重置  
- 個人 Pro 可對外「允許 AI 推薦」（僅企業）  
- **產品路徑的 Dev 登入／測試用方案與 seed_org 進場**（可留測試 API，不進正式 UI）
