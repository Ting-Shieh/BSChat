## 1. 模組目的（一句話複習）

Pro / Enterprise 專屬：將個人職場公開資料（LinkedIn URL 或 People Search）經 LLM 整理為結構化「可能負責…」職責摘要，比 Free 的 M3 名片推估更準。
**對應 PRD**：§11.2.1 C、§12.5、DDR-74/75/76（正向追溯 ✅）。

## 2. 在系統架構裡的位置

- **我依賴誰（上游）**：
  - M1 — `person_enrich_mode` / quota entitlement gate（API + worker 雙檢查）
  - M2 — handoff 帶 `linkedin_url`
  - M6 — 唯讀 `main_products` 作 LLM 輸入
  - 外部 — People / LinkedIn 資料來源（Pilot 選型；目前 `person_search_provider=mock`）
- **誰依賴我（下游）**：
  - M3 index — `person_scope` 拼進 search_text
  - M5 — `from_search` 觸發；`match_reason` 引用 person 欄位（gate：`person_scope_confidence ≥ 0.75` 且 `match_score ≥ 0.8`）
- **對外介面**：
  - `POST /contacts/:id/person-enrich`、`/status`、`/confirm`、`/reject`（Pro+；Free → 403 `PERSON_ENRICH_NOT_ALLOWED`）
  - Worker task `person.enrich`（trigger：`url_auto` / `manual` / `from_search`）
  - `GET /contacts/:id` → `sections.ai_inferred.person_scope`
