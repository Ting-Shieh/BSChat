# BSChat PRD v3 — 團隊聯絡人情報工具（收斂重定位版）

> **文件性質**：本版為 2026-07 產品重審後的**收斂重定位**，取代 v2 的「範圍」與「persona」層。
> **狀態**：草案（待 dogfood 驗證）。核心價值主張、DDR-5/6/7、負面情境 N-01~08 自 v1/v2 **延續有效**；本版**收斂範圍、改變 persona 與名片可見模型**。
> **確認日期**：待自家團隊 dogfood 驗證後回填。
> **一句話**：把團隊手上一整池名片，AI 自動讀懂，讓團隊在需求出現的當下反向找到對的人與公司。

---

## 0. 為什麼有 v3（重審結論）

v2 把 BSChat 當成「**一個業務個人**搜自己抽屜」的消費者 App。2026-07 重審揭露真實處境：使用者**經營一間公司**，底下有業務團隊（含**業務助理**）與**採購**。真實痛不是「一個人找不到名片」，而是：

- **業務助理**每天**手工用表格**整理名片（特質、職稱、經歷、合作可能性、高階者喜好）→ 大量工時。
- **採購**要**拓展新的採購合作商**，手上名片不夠用 → 退回 Google 土法煉鋼。

→ BSChat 重定位為 **AI 團隊聯絡人情報工具**：**v1 先給自家公司團隊用（dogfood）**，**願景是賣給像這樣的業務團隊 / 公司**。

**與 v2 的關係**：
| v2 假設 | v3 修正 |
|---------|---------|
| Persona＝單一業務個人 | **團隊**（主管 / 業務 / 業務助理 / 採購）共用一池 |
| 名片預設私密·單機 | **兩層可見模型 + 公司政策旋鈕**（v1 預設團隊共享）|
| 秘書/CRM dossier＝範圍外 | **AI 自動 dossier＝v1 核心價值**（取代助理手工表格）|
| Pro 跨池 / 企業公開目錄為主線 | **外部供應商發現池（雙邊）移到 Later / 願景** |
| 冷啟動靠簽企業客戶 | **自家公司同時是第一供給＋需求，冷啟動消失** |

**延續不變**：DDR-5（不固定 profile、對話式即時意圖）、DDR-6（看懂公司＋反向找商機）、DDR-7（夠用＝公司產品＋個人職責）、負面情境 N-01~08、AI 透明三區設計原則。

## 1. Persona（團隊角色，共用一池）

| 角色 | 目標 | 現況痛 |
|------|------|--------|
| **主管 / 老闆**（首位使用者本人） | 看整個團隊的人脈資產、找商機 | 資產散在各人腦袋與各自表格 |
| **業務** | 收錄名片、顧關係、找合作 | 收了不整理、想不起誰做什麼 |
| **業務助理** | 把名片整理成可用情報 | **手工表格**（特質/職稱/經歷/合作可能性/喜好）＝高工時 |
| **採購** | 拓展新的採購 / 供應合作商 | 手上名片不夠 → 靠 Google |

**首要驗證對象**：業務助理（工時是否下降）、採購（是否棄用 Google）。

---

## 2. 核心價值（v1 must-have）

### 2.1 殺手時刻（延續原型）
需求出現當下 → 一句自然語言 → 從**團隊整池**反向撈出對的人/公司 → **商機簡報**（為什麼是他 + 沉睡標記 + 開場白 + 誠實不湊數）。原型：`spec/screens/screen-flow-sales.html`、`screen-1-opportunity-briefing.html`。

### 2.2 AI 自動 dossier（取代助理手工表格）— v3 新核心
收錄後 AI 自動補：
- 公司主要產品/服務（M6，延續）
- 個人負責業務推估（M3，延續）
- **特質 / 過去經歷 / 合作可能性**（新增，對齊助理手工欄位；標來源與信心；可人工覆寫）

> 目的：讓業務助理**不用再手刻表格**。AI 產生初稿 → 人工微調,而非人工從零建。

### 2.3 團隊共享池（見 §3）
一池名片,團隊依政策可見、可共同搜尋。

### 2.4 沉睡喚醒
標記「💤 沉睡 N 個月未聯絡」,撈出被遺忘的商機（延續原型）。

## 3. 名片可見範圍模型（v3 核心決策）

**兩層,一個政策旋鈕：**

- **Layer 1 · 跨公司（絕對,寫死）**：名片永不被本公司以外的人搜到。
- **Layer 2 · 公司內（政策可設,不強制）**：
  - 政策 A｜**團隊共享**（名片＝公司資產）
  - 政策 B｜**個人私有**（僅收錄者可見）
  - 可混合：預設一種 + 個別覆寫

**v1 dogfood 實作紀律**：
- 預設＝**團隊共享**（依自家公司政策）。
- **單一設定,不建權限管理 UI。**
- 但資料模型把「可見範圍」設計成**一個欄位/維度**,未來加公司政策開關不需重寫。

> 這**不推翻** v2「預設私密」——那是 Layer 2 旋鈕轉到「個人私有」。同一個模型的不同預設。

## 5. User Stories（v1）

**US-v3-1（業務助理·核心）**
> 作為業務助理,我要讓 AI 自動把名片整理成含特質/經歷/合作可能性的情報,這樣我不用再手工建表格。
> AC: Given 名片收錄完成, When AI 補全, Then 自動產生 dossier 欄位（標來源/信心）,我只需微調而非從零建。

**US-v3-2（採購·核心）**
> 作為採購,我要用一句話從團隊名片池找出「可能的採購合作商」,這樣我不用只靠 Google。
> AC: Given 團隊池有名片, When 我問「有沒有做 XX 零件的供應商」, Then 回排序結果 + 每筆合作可能性理由。

**US-v3-3（業務·核心）**
> 作為業務,我要用當下需求反向找出團隊池裡對的人,並拿到開場白,這樣沉睡名片能變商機。

**US-v3-4（主管·核心）**
> 作為主管,我要搜尋整個團隊的人脈池,這樣商機不綁在某個業務的腦袋或離職。

**US-v3-5（可見範圍）**
> 作為主管,我要決定名片是團隊共享還是個人私有,這樣符合我公司對「名片是不是公司資產」的政策。（v1: 單一預設；設定 UI 為 Later）

**US-v3-6（信任·延續）**
> 作為任何使用者,我要清楚看到哪些是名片原文、哪些是 AI 補的（含信心）,這樣我不會被錯誤推估誤導。

## 6. 明確不做（Later / 願景）

| 項目 | 為什麼不進 v1 |
|------|--------------|
| **外部供應商發現池（雙邊 / Pool B / 企業公開目錄 M11 / 跨池 M5b）** | 雙邊冷啟動極難；v1 團隊內部池已足以驗證核心 |
| **對外消費者版 / 對外銷售** | 先 dogfood 自家團隊,驗證有效再對外（願景 C）|
| **收款後台 / 分級計費 / 多租戶** | 變現水管,PMF 後才做；v1 自己用不需要 |
| **公司政策設定 UI / 細緻權限矩陣** | v1 單一設定即可,避免過度工程 |
| **喜好/請客備註** | 人工欄位,非核心 AI 價值,排 v1.1 |
| **LinkedIn 官方 API** | 卡審核；先用 Gemini 公開搜尋 + card_inference 過渡 |

## 7. 成功指標（dogfood）

| 指標 | 定義 | 目標 |
|------|------|------|
| **助理工時下降** | 業務助理整理每張名片的時間（前 vs 後） | 顯著下降（Pilot 定量）|
| **助理採用率** | 助理是否棄用手工表格、改用 BSChat | 質性:是/否 + 為什麼 |
| **採購搜尋使用** | 採購每週用反向搜尋找合作商的次數 | > Google 使用頻率 |
| **沉睡名片促成接觸** | 因搜尋撈出的沉睡名片而實際聯絡的數 | Pilot 定量 |
| **團隊搜尋滿意度** | 主觀（結果對不對、值不值得 follow-up）| > 3.5/5 |

## 8. 驗證計畫（= Dogfood,不是問卷）

1. 把 v1 給自家團隊真的用（業務助理 + 採購為主）。
2. 觀察:助理**還會不會開 Excel**？採購**還會不會先開 Google**？
3. 質性訪談:哪裡省了時間、哪裡還不夠、哪裡不信任。
4. 依真實使用回頭調 P0/P1 與是否啟動 Later。
